Annotations (2)

  • comment "Indicates a Model to be satisfied if a target resource is to be considered suitable for some specific purpose. A constraint expresses the idea that if some target resource is to be suitable for some purpose, then it must satisfy a designated minim:Model. A constraint may be associated directly with a target resource as the subject of a minim:hasConstraint statement, or may be associated indirectly via a URI template that is the object of an associated minim:forTargetTemplate statement. The latter form is preferred as it allows a given constraint to be more easily used with multiple resources." ()
